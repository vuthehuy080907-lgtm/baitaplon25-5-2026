Trang Debug

Đoạn Mã Đã Sửa Lỗi Hoàn Chỉnh & Giải Thích Sai Sót

Dưới đây là hàm addCourse(newCourse, list) gốc đã được chỉnh sửa lại các lỗi sai logic và cú pháp theo yêu cầu: 

function addCourse(newCourse, list) {
    // Ép kiểu mssv sang số nguyên để thực hiện phép chia lấy dư (%)
    const mssvInt = parseInt(newCourse.mssv);

    /* * LỖI SAI GỐC 1: Điều kiện gốc ghi `if (mssv % 2 !== 0)` -> sai vì biến `mssv` chưa được định nghĩa độc lập, phải lấy từ `newCourse.mssv`.
      * LỖI SAI GỐC 2: Logic xử lý bị ngược. Đề bài yêu cầu số lẻ chèn ĐẦU bảng (phải dùng unshift), số chẵn chèn CUỐI bảng (phải dùng push).
      Code gốc đang làm ngược lại (Lẻ push, Chẵn unshift).
    */
    if (mssvInt % 2 !== 0) {
        list.unshift(newCourse); // ĐÚNG: Số lẻ chèn vào ĐẦU mảng (Mảng render từ trên xuống đầu bảng)
    } else {
        list.push(newCourse);    // ĐÚNG: Số chẵn chèn vào CUỐI mảng
    }

    // Gọi hàm cập nhật hiển thị lên bảng dữ liệu
    renderTable(list);
}
